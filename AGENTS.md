# AGENTS.md

## Overview

`coding-exam-github-app` (課題用アプリ, "Repo Digger") — a Next.js 16 App Router app that
searches and browses GitHub repositories via GitHub's GraphQL API (`urql` + graphql-codegen).
Uses React 19, Tailwind CSS v4, TypeScript (strict), and pnpm.

## Commands

Package manager is **pnpm** (see `pnpm-workspace.yaml`); do not use npm/yarn despite README examples.

```bash
pnpm dev            # dev server at http://localhost:3000
pnpm build          # production build
pnpm start          # serve the production build
pnpm lint           # Biome check (formatter + linter + assist/organizeImports), read-only
pnpm lint:fix       # Biome check --write (applies safe fixes, incl. import sorting)
pnpm typecheck      # tsc --noEmit
pnpm test           # Vitest unit tests (run mode) — src/**/*.test.{ts,tsx} only
pnpm test:e2e       # Playwright E2E (tests/**/*.spec.ts)
pnpm codegen        # graphql-codegen --watch (regenerates src/lib/gql/)
```

**mise tasks** (`mise task` to list, `mise run <name>` to execute; defined in `mise.toml`):

```bash
mise run download-schema      # download GitHub's public GraphQL schema -> schema.docs.graphql
mise run e2e-test             # pnpm build, then pnpm test:e2e (Playwright against the prod build)
mise run e2e-show-report      # open the last Playwright HTML report
mise run precommit            # run all pre-commit hooks via prek (prek run)
mise run precommit-validate   # validate the prek config (prek validate-config)
mise run workflow-check       # pin GitHub Actions / reusable workflows (pinact run)
```

`schema.docs.graphql` is **not committed** (git-ignored). Run `mise run download-schema`
once after cloning to fetch it locally before running `pnpm codegen`. The generated
`src/lib/gql/` output **is** committed, so `pnpm codegen` is only needed when GraphQL
documents change.

## Architecture & conventions

- **App Router** under `src/app/` — `layout.tsx` is the root layout (sets Geist fonts via `next/font`, base flex column body); `page.tsx` is the home route. Add routes as nested directories under `src/app/`.
- **Path alias**: `@/*` maps to `./src/*` (tsconfig).
- **Styling**: Tailwind CSS **v4** — configured through `src/app/globals.css` and `postcss.config.mjs` (`@tailwindcss/postcss`), not a `tailwind.config.js`.
- **Lint/format**: Biome (`biome.json`), not ESLint/Prettier. 2-space indentation; Next.js and React recommended domains enabled; `noUnknownAtRules` disabled (for Tailwind `@` directives). VSCode is set to format with Biome on the project.
  - `biome check` (= `pnpm lint`) runs three things: **formatter + linter + assist**. Import sorting is an **assist** action (`source.organizeImports`), which `biome format` does **not** do — that's why this project standardizes on `biome check` (not `biome format`) for both `pnpm lint` and the pre-commit hook. Auto-fix with `pnpm lint:fix` (`biome check --write`).
- **GraphQL data fetching**: queries/fragments live colocated with the code that uses them; `graphql()` from `@/lib/gql` is the typed-document helper (generated by codegen). Data is fetched in **Server Components** via `getClient()` from `src/lib/github/client.ts`, which is `@urql/next/rsc`'s `registerUrql` (request-scoped urql client; cached per request via React `cache`). The `src/lib/github/*` functions accept an injectable `client` option for tests. Client Components receive plain serializable data, not masked fragment refs (fragments are unmasked server-side).

## Testing

- **Unit/integration**: Vitest. Files are `*.test.{ts,tsx}` under `src/`. Default env is `node`; component tests opt into DOM with a top-of-file `// @vitest-environment happy-dom`. `vitest.config.ts` restricts `include` to `**/*.test.{ts,tsx}` and excludes `tests/**`, so E2E specs are never run by `pnpm test`.
- **E2E**: Playwright. Specs are `tests/**/*.spec.ts` (`testDir: "./tests"`), run with `pnpm test:e2e`. Playwright starts a GraphQL Yoga mock (`tests/mock/`, `pnpm run mock`) on `:4000`; it loads the real `schema.docs.graphql` (git-ignored — run `mise run download-schema` first), so the app must point at it via `API_ENDPOINT=http://localhost:4000` (set in CI by the workflow, locally via `.env.local`).
- **Pre-commit (prek)**: runs Biome format, `tsc`, and **`pnpm test`** (see `prek.toml`). prek stashes unstaged changes before running hooks, so config relevant to the hooks (e.g. the Vitest `exclude`) must be committed/staged to take effect.
